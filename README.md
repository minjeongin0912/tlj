## 문제 해결 과정

**1. 이미지 경로 오류**
- 현상: 로컬에서는 이미지가 표시되었으나 GitHub Pages 배포 후 깨짐
- 원인: 경로에 대소문자 불일치, 상대경로 기준 오류
- 해결: 파일명과 경로를 일치시키고 `./image/` 기준으로 통일

**2. CSS 레이아웃 틀어짐**
- 현상: 푸터 영역이 콘텐츠 위에 올라오거나 의도한 위치에 오지 않음
- 원인: float 또는 position 속성 해제 누락
- 해결: clearfix 패턴 적용 및 구조 재확인

**3. 슬라이더 동작 불안정**
- 현상: 빠르게 클릭 시 이미지가 중첩되거나 순서가 어긋남
- 원인: 이벤트 중복 등록
- 해결: 조건 분기 추가 및 클릭 중 재클릭 방지 처리

## 회고

처음으로 기획부터 마크업, 스타일, 스크립트까지 혼자 완성한 프로젝트다.  
배울 때는 쉬워 보였던 `float`, `position`, `z-index`가 실제로 쓰이면 예상과 다르게 동작해서 당황했다.  
특히 이미지 경로 문제는 로컬과 배포 환경의 차이를 처음 경험한 계기가 됐다.

다음 프로젝트에서는 `flex`와 `grid`를 적극적으로 써보고,  
JavaScript도 함수 단위로 분리해 가독성을 높이는 방향으로 발전시키고 싶다.
